# Чек-лист "Крестики-Нолики"

## Задание 1. Создание структуры проекта

- Каждый слой является отдельным модулем.
- Структура проекта имеет следующие слои: web, domain, datasource, di.
- Слой web включает в себя как минимум пакеты model, module, route, mapper.
- Слой domain включает в себя как минимум пакеты model, service.
- Слой datasource включает в себя как минимум пакеты model, repository, mapper.

## Задание 2. Реализация domain-слоя

- Описана модель игрового поля в виде целочисленной матрицы.
- Описана модель текущей игры, у которой есть UUID и игровое поле.
- Описан интерфейс сервиса, у которого есть следующие методы: 
    - метод получения следующего хода текущей игры алгоритмом «Минимакс», 
    - метод валидации игрового поля текущей игры, 
    - метод проверки окончания игры.
- Модели, интерфейсы, реализации находятся в разных файлах.

## Задание 3. Реализация datasource-слоя

- Реализован класс-хранилище для хранения текущих игр.
- В качестве средства хранения использованы потокобезопасные коллекции.
- Описаны модели игрового поля, текущей игры.
- Реализованы мапперы domain<->datasource.
- Реализован репозиторий для работы с классом-хранилищем, у которого есть следующие методы: 
    - метод сохранения текущей игры, 
    - метод получения текущей игры.
- Создан класс, реализующий интерфейс сервиса и принимающий в качестве параметра репозиторий для работы с классом-хранилищем.
- Модели, интерфейсы, реализации находятся в разных файлах.

## Задание 4. Реализация web-слоя

- Описаны модели игрового поля текущей игры.
- Реализованы мапперы domain<->web.
- Реализован контроллер с использованием Gin, имеющий метод POST /game/{UUID текущей игры}, который отправляет текущую игру 
с обновленным игровым полем пользователя и получает в ответ текущую игру с обновленным игровым полем компьютера.
- Если отправлена некорректная текущая игра с обновленным полем, возвращается ошибка с описанием.
- Есть поддержка нескольких игр одновременно.
- Модели, интерфейсы, реализации находятся в разных файлах.

## Задание 5. Реализация di-слоя

- Реализован класс Container, в котором описывается граф зависимостей.
- Класс-хранилище определен в качестве singleton.
- Определен репозиторий для работы с классом-хранилищем.
- Определен сервис для работы с репозиторием.
